# VERIDIS — Project Context & Architecture

## What This Project Is

VERIDIS is a smart campus recycling Flutter app for **Academic City University** students and staff (Ghana). Users deposit plastic/glass bottles into a physical sorting machine on campus. The app:
1. Links the user to the machine via QR code scan
2. Guides them through depositing multiple bottles
3. Shows real-time classification and weight per bottle
4. Credits earnings to an in-app wallet
5. Allows withdrawal to MTN Mobile Money

---

## Environment

- **Framework:** Flutter (Dart 3.10.7+)
- **Platforms:** Android, Web (Chrome) — iOS excluded (no Xcode)
- **Run command:** `flutter run -d chrome` for development
- **Database:** Cloud Firestore (NoSQL document database, part of Firebase)
- **Auth:** Firebase Authentication (Email/Password)
- **Dependencies:** `mobile_scanner`, `firebase_core`, `firebase_auth`, `cloud_firestore`
- **No real payment API** — wallet is simulated. Admin manually processes MoMo withdrawals.
- **Hardware:** ESP32 microcontroller integration planned but not yet complete

## Firebase / Firestore Setup

### Project
- Firebase project ID: `veridis-app`
- Firestore region: `africa-south1` (Johannesburg)
- Auth provider: Email/Password enabled

### Firestore Collections

**`users/{uid}`**
```
name, email, mobileMoneyNumber, createdAt (Timestamp),
totalWeight (double), totalEarnings (double), totalCo2Saved (double), sessionCount (int)
```

**`sessions/{sessionId}`**
```
userId, machineId, startTime, endTime (Timestamps),
bottles: [{materialType, weightKg, earnings, co2Saved, scannedAt}],
totalWeight, totalEarnings, totalCo2Saved, bottleCount
```

**`walletTransactions/{txId}`**
```
userId, type ("credit"|"withdrawalRequest"), amount, description, isPending (bool), timestamp
```

### Composite Indexes (already created)
- `sessions`: `userId` ASC + `startTime` DESC
- `walletTransactions`: `userId` ASC + `timestamp` DESC

### Security Rules
Production mode — all writes require auth and ownership check. Rules deployed in Firebase Console.

## Auth Flow

- `main.dart` uses `StreamBuilder<User?>` on `FirebaseAuth.instance.authStateChanges()`
  - Waiting → green loading screen
  - Logged in → `HomeScreen` + loads sessions & transactions from Firestore
  - Logged out → `LoginScreen`
- Sign out clears local `SessionService` and `WalletService` data then calls `FirebaseAuth.signOut()`

## Data Models

### `MaterialType` (enum, in recycling_session.dart)
```dart

```

### `BottleItem` (bottle_item.dart)
```dart
class BottleItem {
  final MaterialType materialType;
  final double weightKg;
  final double earnings;
  final double co2Saved;
  final DateTime scannedAt;
}
```

### `RecyclingSession` (recycling_session.dart)
```dart
class RecyclingSession {
  final String id;
  final String machineId;
  final DateTime startTime;
  final DateTime? endTime;
  final List<BottleItem> bottles;

  // Computed:
  double get totalWeight
  double get totalEarnings
  double get totalCo2Saved
  int get bottleCount

  // Static helpers:
  static double earningsFor(MaterialType, double kg)
  static double co2For(MaterialType, double kg)
}
```

**Earnings rates:**
- `kPlasticRateGhs = 0.30` GHS/kg
- `kGlassRateGhs = 0.20` GHS/kg
- `kPlasticCo2PerKg = 2.5` kg CO₂/kg
- `kGlassCo2PerKg = 0.5` kg CO₂/kg

### `WalletTransaction` (wallet_transaction.dart)
```dart
enum TransactionType { credit, withdrawalRequest }

class WalletTransaction {
  final String id;
  final DateTime timestamp;
  final TransactionType type;
  final double amount;
  final String description;
  final bool isPending;
}
```

## Hardware Pipeline (simulated — 5 steps, ~4.1s total)

| Step | Hardware | Simulated delay |
|---|---|---|
| detecting | Proximity sensor | 0.8s |
| capturing | HuskyLens AI camera | 1.0s |
| classifying | LDR + IR sensors | 1.0s |
| weighing | Load cell + HX711 | 0.8s |
| sorting | Servo motor gate | 0.5s |

**ESP32 integration (planned):** When hardware is ready, ESP32 will write bottle data directly to Firestore via REST API. The `sessions` Firestore schema is already designed to receive this data.

## Key Design Decisions

1. **No button on the machine** — user taps "Done" or "Add Another" in the app.
2. **One session = multiple bottles** — `RecyclingSession.bottles` is a list.
3. **Simulated wallet** — no Paystack/MoMo API. Admin manually sends MoMo after withdrawal request.
4. **Fire-and-forget Firestore writes** — Firestore offline persistence handles retries automatically.
5. **`_pages` is a getter (not const)** in `HomeScreen` so tabs always rebuild with fresh data.
6. **`hide MaterialType`** — Flutter's `material.dart` exports a conflicting `MaterialType`. Screens importing both use `import 'package:flutter/material.dart' hide MaterialType;`.
7. **Theme via `AppTheme.theme`** — never add raw `Color(0xFF...)` literals in screens, always use `AppColors.*`.
